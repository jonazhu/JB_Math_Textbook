0. Introduction#

In this chapter, we will introduce the concept of data analysis and how to write and use Python programs to explore data for analytical purposes. The organization of this chapter is as follows: First, we will begin with an overview of data, describe the background of processing data, and explore various data types. Then, we will show how the Python programming language is used to prepare the data for analysis, including data importing and selection. Next, we will introduce common data operations used for statistics and analysis. Additionally, advanced data operations with applications will also be presented. Finally, we will give practical examples to demonstrate the process of data analysis for a variety of topics.

1. Exploring Data and Data Analysis#

There are several data-related research areas. A database researcher is interested in how data is stored and managed, whereas a data scientist focuses more on the topics such as methods of processing massive data, analyzing data from various sources, interpreting analyzed data, and visualizing the analytical results. Data analysis, in particular, has drawn attentions to statisticians, computer scientists, business people, health professionals, or even philosophers and artists.

Traditionally, data analysis was thought to be done by mathematicians only. Scholars often considered it in the domain of statistics. However today, there are new aspects and challenges that motivate us to think of data analysis as a broad and multidisciplinary area. We develop and utilize a variety of tools including computer programs to perform data operations. Due to the scope of this chapter and this book, we only describe a few facts of data and its related challenges in the era of modern data analysis.

At present, scholars are dealing with large-scale data everywhere. For example, the Internet today has billions of webpages. According to the conservative estimates, there are at least three hundred billion emails that are sent daily. The need of analyzing such a great amount of data effectively has driven scientists to develop well-designed methods and to constantly update the hardware to increase the computing power.

In today’s real world, the speed of data generation is fast. Even if the data is from same source, numbers that are collected in the past can be very different from them at present. At the early stage of the COVID-19 pandemic, researchers and decision makers realized that the reported data of disease infections fell behind. The collected data for the daily reporting largely reflected what had happened in the past rather than what was happening.

Data analysts are presented with structured, semi-structured, or unstructured data. One example of a structured dataset may be from a professor’s grade book. Students’ names and grades of all assignments are presented completely and accurately in a well-organized and cross-checked spreadsheet/table. Less or no additional effort of data “re-structuring” needs to be made. However, data from an ordinary person’s diary can be unstructured. The personal note has various lengths, covers different and unrelated topics, and is likely handwritten. Extracting data for an analytical process requires tremendous amount of data pre-processing work.

Just like water as the resource for people, agriculture, and industry, data is everywhere. However, just like water, the origin and the quality of data matter. The prevalence of data does not imply that all kinds of data can be used without scrutiny. An online survey result that was collected anonymously and assembled by a controversy business entity may be a less reliable but more biased source for data analytics. The clinical trial data submitted by the responsible medical professionals and peer reviewed by a reputable research group is thought be more trustworthy for academic purposes. The extent of data pre-processing depends on the source and the collection method of data.

There are various features of data due to the difference of dataset size, generation method, structure, and its sources. For analyzing data, several methods and challenges exist, too. The direction of data analysis can be predictive or descriptive:

For a predictive data analysis, one uses existing data to predict unknown and future data. For example, a credit card company can analyze the history of a user’s credit card use to detect the potential fraud and possibly unauthorized transactions. An economist may analyze data from multiple sources to model the likelihood of a recession in the next few weeks or months. A climate scientist is able to create a model to predict the temperature trend from a region based on the historical pattern of the climate data.

For a descriptive data analysis, one performs analysis on available data in order to discover and better understand the features and patterns of data from the dataset. For example, a genealogist may use a program to analyze the people’s face to tell how two individuals’ look is similar to or different from each other. A business sales team may perform a market basket analysis from millions of transactions to discover the sets of frequently brought items.

The choice of a data analysis method depends on not only the objective of analysis but also the type of data. An attribute of the data is here defined as a property or a feature of such kind of data. Generally speaking, there are four common types of data according to their attributes [cite: S. S. Stevens]. Nominal: it is the type that is usually the names or values themselves, such as postal codes or identification numbers. Ordinal: it is the type that is the names or values, and they can be ordered or ranked, such as wine’s quality grades or customer’s satisfaction ratings. Interval: it is the type that has the feature of ordinal data, i.e. ordered, and the difference of data values is more explainable, such as calendar dates. Ratio: it is the type that is ordered and can be meaningfully multiplied or divided, such as the distance between two locations.

Data operations must be suitable for the specific data types. We can say that iron is about 7.8 times heavy of water (the density of iron is 7.87 g/m3; the density of water is 1 g/m3). However, it is wrong to do a simple arithmetic sum operation by saying the density of a mixture of iron and water is 8.8 (7.8+1) times heavy of water. In biology, a DNA molecule is composed of four types of bases: adenine (A), thymine (T), guanine (G), and cytosine (C). These four bases are paired as A to T and G to C. However, we cannot just “compare” them to determine which base is “larger” or “precedent” of another, in the usual way as we do with natural numbers.

In the next part of this chapter, we will cover data analysis topics from executing common statistical tasks to building specific models for further analysis. We will demonstrate to the readers how to write Python programs relevant to the theoretical aspect of analysis. Moreover, we will discuss the benefits of utilizing computer programs when the data is processed with the help of Python libraries.

2. Pre-processing and Post-processing Data: Importing and Exporting Data, and Data Queries#

For the complete process of data analysis, programmers usually follow the sequence of:

1 Reading data from a file or from the user input  2 Processing loaded data, including re-organizing data, computations, and generating results  3 Outputting data to the computer screen or writing the output to files.

The first step is pre-processing the data, that is, to prepare the “clean” data for analysis from the raw data. This step usually involves the loading of a dataset that is imported from databases, files, or the user inputs. The raw data can take various forms. As we have mentioned in the previous section, the data can be more or less structured. In this chapter, we will focus on the processing of structured datasets.

A structured dataset, according to this book’s definition and its applications, is the one that is:

  1. linear, such as a collection of records with the same type, i.e. 1-dimensional data, or

  2. table-like, such as the records organized with complete information of rows and columns, i.e. 2-dimensional data, or

  3. cubiod-like, such as data points in a space, represented by three values, x, y, z, i.e. 3-dimensional data).

Since the computer screen is “flat”, i.e. 2-dimensional, and all data must be properly structured and saved into the memory space, we still need to convert the 3-dimensional data in a form that fits into a 2-dimensional data’s view. There are several techniques that do such kind of 3D-to-2D conversion, but this topic is currently beyond the scope of our book. Therefore, we only introduce the methods of processing a file that is already saved with one dimension or two dimensions.

2.1 One-dimensional data processing#

For a one-dimensional dataset, a Python program can read the data from the user input one after one. Ultimately, the program may save all user-inputted data into a data structure such as a list ([ ]).

i) The program asks the number of inputs by the user print(“How many numbers to input”) num_inputs = int(input()) # The input() function takes a string by default. Converting to int type. data_list = [] # Initial list is empty

ii) The user inputs records one after one. All will be saved into data_list print(“Enter each record. Press Enter to continue.”) for i in range(num_inputs): user_data = input() data_list.append(user_data)

iii) Output the list that contains all previously inputted data print(data_list)

Another situation to read the data is from a file. For the convenience of processing, we assume that such a file is a plain text file and it only contains relevant data for calculations (i.e. no data labels, titles, informational fields, or any empty spaces or lines). Furthermore, we assume that all records are stored in one line where the two records are separated by a single white space (see example 1), or that they are stored in multiple lines where each line contains one record (see example 2).

Exampke 1#

[program example 1: one-line, white space]

[figure: one-line file]

i) First, open the file that contains all records; store the content to the object data_file_1. data_file_1 = open(“datafile1.txt”, “r”)

ii) Then, read the data_file object for the first (also the only and the last) line and save it to a string. one_line = data_file_1.readline()

iii) Next, split the string into several records and further save the records into a list. iv) Note: the white space is the separator in this program. data_list = one_line.split()

v) Finally, output the data from the saved list, data_list. print(data_list)

vi) Close the file. data_file_1.close()

Explanation:

The file open() function in this code example takes two arguments. The first one is the file name. The second one is the file processing mode. “r” here means read. The readline() function outputs only one line of data from a file if it is used once.

The split() function is used to split data based on the separator. If nothing is specified in the function’s argument, the default separator is white spaces.

As a good programming practice, programmers should close the file before the end of the program by explicitly writing the file closing function close().

i) First, open the file that contains all records; store the content to the object data_file_2. data_file_2 = open(“datafile2.txt”, “r”)

Explanation:

The readlines() function differs from the readline() function. The program will continue to read another line till the end of the file. The for loop is used to iterate each line. The print() function inside the for loop will print each line’s content except the line break character at the end of each line.

This program treats each line as a string. The code line[:-1] means that the program does slicing of the string so that the new string contains all character from the first to the second last one. For a plain-text file, an end user or an ordinary reader sees the whole file with several lines. These lines are technically separated by the line breaks, or the so-called “enter” keys.

In most programming languages including Python, such a line break character is represented by \n. That \n is not a part of “data”, thus should be removed. Please note that we must ensure that the last record or line of the file is also followed by a line break (\n) or that record will not be processed correctly.

2.2 Two-dimensional data processing#

An efficient way of storing the data in a table (i.e. two-dimensional data) is using a file format called Comma-Separated Values (CSV). Such a file is named like transaction_report_2023.csv . A typical .csv file looks like the following:

[figure]

On occasions, a .csv file may include a list of titles or attribute names in the first line while others may not. Such a title/attribute line is called “header” in our file processing applications. Unlike the real-world data table where each line of record is presented first with a record number or an index value, the .csv file usually does not have such a number as the first column information of the data. All data records begin with the actual data in each line (see example below).

[figure]

After we have obtained the data file and put it into a file system, we can use a popular Python library “pandas” to load and process such a file. The step of reading a .csv file with the Python’s Pandas library is as follows:

i) Import the pandas library. You can optionally rename “pandas” as “pd” for convenience.

ii) Read and load the .csv data from the specified file location into the program’s memory space, and save the data into a format called “DataFrame”.

iii) Operate on the object with the DataFrame format.

import pandas as pd my_data = pd.read_csv(‘data.csv’) print(my_data)

On line 1, we import the pandas library, but we may not want to write the complete name of pandas every time when we use it. Therefore, we use the alias pd to denote pandas.

On line 2, my_data is the object with the DataFrame format. It is used to be processed by the Pandas library according to the row and column information. pd.read_csv is the function from the Pandas library to read the .csv file. Enclosed in the function’s parameter ‘data.csv’ is the file name (data.csv).

If your Python’s workspace and the data file are in the same directory/folder, no further action is needed from this part. However, if the data file is stored elsewhere, we must specify the location of such a file in the Python code, such as:

my_data2 = pd.read_csv(‘user/math_teacher/Desktop/project2/data2.csv’)

or

my_data3 = pd.read_csv(‘C:\Users\Math_Student\Desktop\Project3\data3.csv’)

Note that the Python code for specifying a file location depends on the operating system the user is using. The first form is usually suitable for a *nix system, including MacOS, most Unix and Linux systems, where a forward slash (/) is used to go into a sub-directory. The second form is often seen in a system where a backward slash () is used to go into a sub-directory, such as Microsoft Windows or DOS. Here, two backward slashes are used because Python processes the special character \ with an escape character . The first \ is for the formatting purpose to indicate that there is a special character coming next and the second \ is the actual character to be read.

As the file name has suggested, the file contains comma-separated values because commas (,) are commonly used to separate the columns. However, files can still be processed by the Python program if the columns are separated by a symbol other than a comma. Such a symbol may be a white space ( ), a semi-colon (;), or any other symbol.

In order to read such kind of file and process the column information correctly, we change the parameter part “sep=X” to indicate a different separator. “X” is replaced by the separator.

For example,

my_data4 = pd.read_csv(‘data4.csv’, sep = ’ ‘)

or

my_data5 = pd.read_csv(‘data5.csv’, sep = ‘;’)

From the above examples, the first one is used to process a file where columns are separated by a single white space. The second one is by a semi-colon. If no parameter is specified, this read_csv function will process the file that treats the comma as the separator by default.

i) Read data

The first step in our data analysis process is reading the raw data from a file and loading it into the Panda’s DataFrame. Now, we assume that this step has been completed. Our next step is to operate on the data that is stored in the DataFrame for analysis. In this section, we will show the process of several basic statistical operations in Python. We operate data based on a dataset from a .csv file. The dataset we are using for demonstration is the world population data from the World Bank of the United Nations:

https://api.worldbank.org/v2/en/indicator/SM.POP.TOTL?downloadformat=csv

After we have downloaded the file, we need to extract the .zip package, which contains two meta-data files (for explanation purposes) and one population data file (API_SP.POP.TOTL_DS2_en_csv_v2_5454896.csv), which we will use for analysis.

After removing the unnecessary lines as described, we save that file with a better-recognized name, for example, world_population.csv .

ii) Select data

A dataset is often so large that we do not want to display everything from it. Sometimes, we just want to take a glance at the structure of the dataset by viewing the first few rows. The following code is executed to display the first 5 rows of data.

population_raw_data.head()

[figure]

If we do not put any number parameter in the head() method, the default display of the DataFrame is with the first 5 rows. If a specific number X is put as the parameter, e.g. population_raw_data.head(3), the first X (3) rows will be displayed.

Our analysis does not need all of the data. Therefore, we may select a few rows and/or columns and present the data as needed.

Particular rows may be selected for displaying purposes. For example, we may be interested in knowing the population values for a specific country or multiple countries, we can write the following code to select and display the related data.

country_name = “Ethiopia” country_selected = population_raw_data.loc[population_raw_data[‘Country Name’] == country_name] print(country_selected)

[figure]

Explanation: The above code is for selecting a table’s row(s) based on conditions. In the second line, we have used the .loc expression to locate the rows. One can use it such as .loc[3] or .loc[10:20] where there are number values in the squared brackets to select rows according to their indices of the table. In our code, since we want to specify a selection of row for the country Ethiopia, we apply the expression population_raw_data[‘Country Name’] == country_name . The double equal sign == tells the program that we will select the row where there is a country name “Ethiopia” in the column Country Name.

We do not always have to use == for the selection of a row. For instance, we may want to look at all countries and regions whose population is over 1 billion in the year of 2000. The following code does this job. Notice that > is used in the conditional expression.

population_size = 1000000000 regions_selected = population_raw_data.loc[population_raw_data[‘2000’] > population_size] print(regions_selected)

[figure]

In order to select certain columns and display the data that are associated with those columns, we can write the following code:

data_year1960 = pd.DataFrame(population_raw_data, columns = [‘Country Name’,‘1960’]) print(data_year1960)

[figure]

Explanation: The above code uses Panda’s DataFrame function to “create” another DataFrame from the original one. The new DataFrame contains two columns only, one as ‘Country Name’ and the other as ‘1960’. data_year1960 is therefore also a DataFrame type.

One important thing to note is that from a computer science’s perspective, the index number for counting or for a collection of data starts with 0, not 1. As people generally say we read the first row, this row is reflected as row #0. This offset-by-one convention is applied almost everywhere in computing, including throughout the Python language.

We should also note that Python can automatically recognize the data types from the data. When the Python program detects a number type, such as the population value, it automatically converts the whole number (i.e. integer) into a number with the decimal point (i.e. floating point number). Although the data that is represented with a floating point type seems to be unnecessary in this specific dataset, it may be helpful in certain calculations. Nevertheless, Python treats the two numbers, one as a whole number and the other as the number with the same whole number part plus .0, e.g. 1.0 and 1, equivalent in their numerical value.

Sometimes, the raw dataset contains fields that are without any number but with a space or nothing. Those fields that are supposedly filled in with a number-typed data will show NaN, which means “Not a Number”.

Option 1: Using Python’s built-in functions.

The Python programming language itself carries multiple functions that can be used directly. These functions are called built-in functions in that a user does not need to explicitly write any code to import libraries and their functions from an external source. Usually, these functions are the ones that a programmer needs to use very frequently and basically, such as print(), int(), str(), etc. Some of the built-in functions are for mathematical calculations, such as max() and min().

Here, we demonstrate the following built-in functions in Python, abs(), max(), min(), round(), sum():

abs(): This function outputs the absolute value of an input number.

abs(3.0)

3.0

abs(-102.4)

102.4

abs(0.0)

0.0

max(): This function outputs the largest element in an iterable data collection, e.g. a list.

list_a = [4.04, 10.27, 12.19, 5.25, 1.01] max(list_a)

12.19

min(): This function outputs the largest element in an iterable data collection, e.g. a list.

list_b = [3.14, 15.92, 6.53, 5.89, 7.93] min(list_b)

3.14

round(): This function outputs the round-off value of a floating-point number.

round(10.0001)

10

round(9.9997)

10

round(-0.01)

0

sum(): This function sums up the element values of a list and outputs the result.

list_s = [1, 3, 5, 7, 9] sum(list_s)

25

There are advantages and disadvantages of using Python’s built-in functions for basic data analysis. On one hand, since no external library needs to be imported, the program is relatively easy to maintain. The use of these built-in functions is also relatively easy. On the other hand, there are a limited number of functions available to do statistical analysis. Existing built-in functions may not suitable to use for a uncommon data type unless a data type conversion is done.
